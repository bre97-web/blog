Transition from Vue to Angular

从Vue过渡到Angular.

  • angular
  • vue

Sat May 11 2024 00:00:00 GMT+0000 (Coordinated Universal Time)

由于本人在Angular之前学习过Vue, 本文将会以”从Vue转向Angular”的角度介绍Angular.

创建项目

Angular与Vue很大的不同点在创建项目上出现了. 编写Vue时几乎所有时刻都可以自己安装依赖而不需要cli, 仅仅需要vue, vite, typescript这三个依赖就可以启动初始项目. Angular项目的官方依赖没有过度耦合, 包很多, 通常不会手动一个一个安装依赖.

npm i -g angular-cli

ng new angular-project

组件

Angular中的组件与Vue的有很大的不同:

CompartionAngularVue
组件定义方式通过@Component和class定义通过SFC或defineComponent(或无类型提示的{})定义
组件使用方式通过selector使用组件通过import或components选项使用组件
输入参数通过@Input定义通过defineProps或props声明
输出事件通过@output定义, 通过调用它的emit(...)激活通过defineEmits或emits声明
依赖注入通过Service层和inject注入通过provide和inject注入

创建组件

可通过angular-cli来自动创建组件相应的文件:

ng generate component Button

或者手动地创建需要的文件:

File NameDescriptionOptional
*.component.ts组件的具体定义Required
*.component.spec.ts测试文件Optional
*.component.css样式Optional
*.component.html结构Optional

其中*.component.css, *.component.html, *.component.spec.ts是可选的, *.component.ts是必须的.

定义组件

Angular的组件通过TS装饰器@Component定义, 它接收两个必要参数和其它可选参数:

参数描述可选项
selector选择器, 作为使用该组件时的标签名必须
template or templateUrl用于渲染DOM的HTML模板必须
style, styleUrl or styleUrlsCSS可选
imports依赖可选
………

如下代码创建了一个Button组件:

import { Component } from '@angular/core';

@Component({
  selector: 'app-button',
  template: `
    <button class="filled">I am a button</button>
  `,
  style: `
    button.filled {
        background-color: blue;
        color: white;
    }
  `,
  standalone: true,
})
export class ButtonComponent {
}

使用组件

使用组件的步骤:

  1. 引入组件
  2. 将需要的组件放在imports中
  3. 在模板中使用组件

Angular不把import的名称作为组件标签名, 而是把selector作为组件标签名.

// Angular

/**
 * @file home.component.ts
 */
import { Component } from '@angular/core';
import { ComponentA } from './component-a.component';

@Component({
  selector: 'home',
  standalone: true,
  imports: [ComponentA],
  template: `
    <component-a></component-a>
  `
})
export class HomeComponent {

}

/**
 * @file component-a.component.ts
 */
import { Component } from '@angular/core';

@Component({
  selector: 'component-a',
  template: ``,
  standalone: true,
})

export class ComponentA {

}

组件树

Angular把匹配的每一个HTML元素创建一个组件实例. 组件实例的HTML模板在与selector匹配的DOM元素中渲染, 这个DOM元素被称为这个组件的宿主元素. 组件渲染的DOM被称为该组件的视图, 多个组件构成组件树.

要点:

  • Angular为所有匹配的HTML元素创建一个组件实例
    • 匹配发生在Angular编译时
    • 运行时更改DOM不影响渲染结果
  • 与selector匹配的DOM元素是该组件的宿主元素
  • 组件渲染的DOM叫视图
  • 多个视图组合后叫组件树

Input

@Input()作为组件的输入参数, 它提供更高的可重用性和变化.

如下代码创建了一个TestProp组件, 接收一个可选参数value, 取值为any:

import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-test-prop',
  standalone: true,
  imports: [],
  template: `
    <p>
      PROP VALUES: {{ value }}
    </p>
  `
})
export class TestPropComponent {
  @Input() value: any;
}

这是提供参数到TestProp组件的方式:

<section>
    <app-test-prop value="123"></app-test-prop>
</section>

可选参数

@Input()默认为可选参数.

必须参数

向@Input()提供required参数:

@Input({
    required: true
}) value: any;

类型检查

由于Angular天生基于Typescript, 它直接支持类型检查:

@Input() isOpen!: boolean;

<section>
    <app-test-prop [isOpen]="true"></app-test-prop>
</section>
